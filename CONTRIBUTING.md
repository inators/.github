# Contributing

Thanks for your interest in contributing to inators projects!

## Getting Started

1. Fork the repository
2. Create a feature branch from `main`
3. Make your changes
4. Submit a pull request

## Pull Requests

- Use a clear, descriptive title following [conventional commits](https://www.conventionalcommits.org/) (e.g., `feat: add user authentication`)
- Fill out the PR template completely
- Ensure CI passes before requesting review
- Keep changes focused â€” one feature or fix per PR

## Commit Messages

We use conventional commits enforced on PR titles (squash merge). Valid prefixes:

`feat`, `fix`, `docs`, `chore`, `refactor`, `test`, `perf`, `ci`, `build`, `revert`

## Code Review

- All PRs require review before merging
- Address review feedback or explain why you disagree
- Be respectful and constructive in reviews

## Reporting Issues

- Use the issue templates provided
- Search existing issues before creating a new one
- Include reproduction steps for bugs

## Questions?

Open a discussion in the Q&A category on the org's [Discussions](https://github.com/orgs/inators/discussions) tab.
